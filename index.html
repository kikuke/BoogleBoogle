🎮 프로젝트 소개

BuggleBuggle은 Allegro5를 기반으로 C언어로 개발된 2D 플랫폼 아케이드 게임입니다. 플레이어는 거품을 발사해 적을 포획하고, 3개 스테이지를 클리어해 최고 점수를 기록합니다.

✨ 핵심 기능

  • 플레이어 조작: 화살표 키로 이동, 위쪽으로 점프, 스페이스로 거품 발사
  • 적 AI: 기본몬스터(지상 이동), 투척몬스터(비행 + 투사체 공격)
  • 거품 메커니즘: 발사 → 부양 → 터짐의 3단계 생명주기
  • 점수 시스템: SQLite3 기반 리더보드 저장 및 조회
  • 3단계 스테이지: 난이도 점진적 상승 (⭐ ~ ⭐⭐⭐)
  • 충돌 감지: AABB 알고리즘 기반 정확한 피직스

🎮 게임 관리자 (game_manager.c)

프로젝트의 중앙 허브로, 모든 게임 상태와 리소스를 관리합니다. Game Manager 없이는 단 하나의 함수도 올바르게 작동할 수 없습니다.

🔑 핵심 함수

  • GAME_MANAGER_UpdateState(): 게임 상태를 업데이트하고 반환
  • GAME_MANAGER_SetGameState(): 새로운 게임 상태로 전환
  • GAME_MANAGER_CheckCollision(): 모든 객체간 충돌 판정
  • GAME_MANAGER_UpdatePhysics(): 모든 객체의 물리 적용
  • GAME_MANAGER_UpdateObject(): 모든 객체의 상태 갱신

🎨 UI & 점수 (score.c)

SQLite3를 이용한 리더보드 시스템입니다.

점수 계산

기본 몬스터: +100점 (처치 시) 투척 몬스터: +200점 (처치 시) 남은 생명: +500점 × 남은 목숨 수 (스테이지 클리어 시) 총점 = Σ(스테이지 1~3 점수)

리더보드

  • 저장: Score_Add() - 이름과 점수를 DB에 저장
  • 조회: Score_Get() - 상위 10위 반환
  • 렌더링: Score_Print() - 점수 화면 전체 출력

🎮 게임플레이 메커니즘

BuggleBuggle은 거품 기반 적 포획 메커니즘으로 독특한 게임성을 만들어냅니다.

🎯 기본 게임플로우

1. 스테이지 로드 └─ 플레이어 스폰: (15, 210) └─ 적 배치: 맵 데이터에서 초기화 2. 게임 중 (INGAME) └─ 매 프레임: ├─ 입력 처리 (키보드) ├─ 충돌 감지 ├─ 물리 적용 ├─ 상태 업데이트 ├─ 렌더링 └─ 스테이지 판정 3. 승리 조건 └─ 모든 적 처치 → 다음 스테이지 4. 패배 조건 └─ 생명 0 → NAME 화면 (이름 입력) 5. 최종 화면 └─ SCORE 화면 (리더보드)

💧 거품 메커니즘

  • 발사: SPACE로 플레이어 정면으로 발사 (속도: 3.0 px/f)
  • 상태 1 - SHOOTING (0~30프레임): 수평 직선 이동
  • 상태 2 - FLOAT (30~300프레임): 천천히 상승 (속도: -0.8 px/f)
  • 상태 3 - POP (300~330프레임): 터짐 애니메이션
  • 충돌: SHOOTING 중 벽 충돌 시 → FLOAT 상태로 변환
  • 적-거품 상호작용: SHOOTING 거품이 적 충돌 → 적 TRAPPED 상태
💡 디자인 의도: 거품은 단순 투사체가 아닌 부양하는 함정입니다. 플레이어는 거품을 이용해 높은 곳의 적을 잡거나 본인의 길을 열 수 있습니다.

👾 적 AI 상태 머신

┌─────────────┐ │ IDLE (시작) │ └──────┬──────┘ │ ▼ ┌─────────────┐ │ MOVE │ ← 기본 상태 │ - 좌우 이동 │ (자동 진행) │ - 점프 시도 │ └──────┬──────┘ │ 거품 충돌 ▼ ┌──────────────┐ │ TRAPPED │ ← 포획 상태 │ (타이머) │ (300 프레임) │ - 카운트다운 │ └──────┬──────┘ │ 타이머 종료 ▼ ┌──────────────┐ │ MOVE (화남) │ ← is_angry = true │ - 2배 속도 │ │ - 공격력 강화│ └──────────────┘

🏃 적 타입별 이동

  • BASIC: 지상에서 좌우 이동, 높이 차이 있으면 점프 시도 (50% 확률)
  • THROW: 비행, 벽 충돌 시 방향 변경, 180프레임 주기로 플레이어에게 투사체 발사

🎯 점수 계산 시스템

  • 기본몬스터 처치: +100점
  • 투척몬스터 처치: +200점
  • 남은 생명(스테이지 클리어 시): +500점 × 남은 목숨 수
  • 조기 클리어 보너스: 없음 (난이도 상승이 보상)

예시 계산

스테이지 1: 6마리 BASIC 처치 = 6 × 100 = 600점 스테이지 2: 3마리 THROW 처치 = 3 × 200 = 600점 스테이지 3: 3마리 THROW + 3마리 BASIC = (3×200) + (3×100) = 900점 남은 생명: 2개 × 500 = 1000점 총점 = 600 + 600 + 900 + 1000 = 3100점

📐 맵 구조

  • 맵 크기: 36×24 타일 (총 864개 타일)
  • 타일 크기: 10×10 픽셀
  • 화면 해상도: 320×240 픽셀 (버퍼) → 960×720 픽셀 (3배 확대)
  • 플랫폼: 맵 데이터에서 1 = 타일, 0 = 공중
  • 맵 경계: 상단(y=10), 하단(y=220)에서 자동 경계 처리

타일 배치 원칙

  • 스테이지 1: 규칙적 플랫폼 배치 (이동 연습용)
  • 스테이지 2: 더 좁은 틈새와 높이 차이 (회피 강제)
  • 스테이지 3: 복잡한 미로 같은 구조 (최고 난이도)
⚠️ 주의: 적 초기 위치가 타일 내부에 있으면 즉시 튕겨나갑니다. 항상 공중(0) 위치에서 스폰하도록 설정하세요.
